## IP 주소 체계와 관련 기술

IP 주소는 네트워크 계층에서 장치를 구분하는데 사용하는 고유 식별자입니다. 초기에는 클래스 기반 IP 주소 체계를 사용하였으나, 현재는 CIDR과 같은 유연한 주소 체계가 사용되고 있습니다. 이러한 IP 주소 체계를 관리하고 할당하기 위해 DHCP와 NAT와 같은 기술이 사용됩니다.

### 1. [클래스 기반 IP 주소 체계](https://www.cisco.com/c/en/us/support/docs/ip/routing-information-protocol-rip/13788-3.html)

IPv4는 처음에 클래스 A, B, C, D, E로 나누어 IP 주소를 관리하였습니다. 각 클래스는 네트워크와 호스트 비트를 고정된 비율로 나누어 사용하였습니다.

| 클래스 | 주소 범위                   | 네트워크 비트   | 호스트 비트 | 사용 목적       |
| ------ | --------------------------- | --------------- | ----------- | --------------- |
| A      | 0.0.0.0 ~ 127.255.255.255   | 8비트           | 24비트      | 대규모 네트워크 |
| B      | 128.0.0.0 ~ 191.255.255.255 | 16비트          | 16비트      | 중규모 네트워크 |
| C      | 192.0.0.0 ~ 223.255.255.255 | 24비트          | 8비트       | 소규모 네트워크 |
| D      | 224.0.0.0 ~ 239.255.255.255 | 멀티캐스트 전용 | 해당 없음   | 멀티캐스트 전송 |
| E      | 240.0.0.0 ~ 255.255.255.255 | 연구 및 예약용  | 해당 없음   | 미래 용도       |

#### 장단점

- 장점: 구조가 단순하여 이해와 관리가 쉬움
- 단점: IP 주소 낭비가 심하고 유연성이 떨어짐

예를 들어, 클래스 A는 16,777,216개의 호스트 주소를 제공하지만, 대규모 네트워크가 필요하지 않은 경우에도 클래스 A 주소가 할당될 수 있었습니다. 이로 인해 네트워크의 크기와 실제 필요에 맞는 유연한 IP 주소 할당이 어려웠습니다.

### 2. [CIDR (Classless Inter-Domain Routing)](https://aws.amazon.com/what-is/cidr/)

CIDR은 클래스 기반 주소 체계의 한계를 해결하기 위해 도입된 방식입니다. IP 주소 뒤에 `/`와 함께 서브넷 마스크의 비트 수를 명시하여 네트워크와 호스트 비트를 유연하게 설정할 수 있습니다.

- 표기법: IP주소/서브넷 마스크 비트 수 (예: 192.168.1.0/24)
- 서브넷 마스크: 네트워크와 호스트 부분을 구분하는 32비트 주소

#### CIDR의 장단점

- 장점:
  - IP 주소 할당의 유연성 증가
  - 라우팅 테이블 크기 감소로 네트워크 효율성 향상
  - 더 세분화된 네트워크 구성 가능
- 단점:
  - 복잡한 서브넷 계산 필요
  - 기존 클래스 기반 시스템과의 호환성 문제 발생 가능

인터넷 서비스 제공업체(ISP)들이 고객에게 다양한 크기의 IP 블록을 할당할 때 CIDR를 사용합니다. AWS에 VPC를 구축할 때도 CIDR 블록을 지정하여 네트워크를 구성합니다.

### 3. [DHCP (Dynamic Host Configuration Protocol)](https://docs.aws.amazon.com/vpc/latest/userguide/VPC_DHCP_Options.html)

DHCP는 IP 주소 및 기타 통신 매개변수를 자동으로 할당하기 위한 네트워크 관리 프로토콜입니다.

#### DHCP 동작 과정

1. DHCP Discover: 클라이언트가 DHCP 서버를 탐색
2. DHCP Offer: DHCP 서버가 IP 주소 제안
3. DHCP Request: 클라이언트가 제안된 IP 주소 요청
4. DHCP Acknowledge: DHCP 서버가 IP 주소 사용 확인

#### DHCP의 장단점

- 장점:
  - IP 주소 관리의 자동화로 관리 부담 감소
  - 효율적인 IP 주소 사용 (동적 할당)
  - 중앙 집중식 네트워크 설정 관리 가능
- 단점:
  - DHCP 서버 장애 시 네트워크 문제 발생 가능
  - 보안 취약점 (DHCP 스푸핑 등) 존재

대부분의 가정용 라우터와 기업 네트워크에서 DHCP를 사용합니다. 이를 통해 인터넷에 연결된 기기들이 자동으로 IP 주소를 받아 네트워크에 연결할 수 있습니다.

### 4. [NAT (Network Address Translation)](https://www.cisco.com/c/en/us/support/docs/ip/network-address-translation-nat/13772-12.html)

NAT는 패킷이 라우팅 장치를 통해 전송되는 동안 패킷의 IP 주소 정보를 수정하여 IP 주소를 다른 주소로 매핑하는 프로세스입니다.

#### NAT 유형

- Static NAT: 1:1로 사설 IP와 공인 IP를 매핑
- Dynamic NAT: 여러 사설 IP를 공인 IP 풀에 동적으로 매핑
- PAT (Port Address Translation): 여러 사설 IP를 하나의 공인 IP에 포트 번호로 구분하여 매핑

#### NAT의 장단점

- 장점:
  - IP 주소 절약
  - 내부 네트워크 보안 강화
  - IPv4 주소 고갈 문제 완화
- 단점:
  - 일부 애플리케이션과의 호환성 문제
  - End-to-End 통신 원칙 위배
  - 네트워크 성능 저하 가능성

대부분의 가정용 및 소규모 기업용 라우터에서 NAT를 사용합니다. 예를 들어, 하나의 공인 IP 주소를 가진 가정에서 여러 기기가 동시에 인터넷에 접속할 수 있는 것은 NAT 덕분입니다.

### 5. [IPv6](https://aws.amazon.com/ko/compare/the-difference-between-ipv4-and-ipv6/#:~:text=The%20Amazon%20VPC%20IPv6%20on,only%20subnets%20and%20EC2%20instances.)

IPv4 주소 고갈 문제를 해결하기 위해 개발된 차세대 인터넷 프로토콜입니다.

- 128비트 주소 체계 사용 (340,282,366,920,938,463,463,374,607,431,768,211,456개의 주소 제공)
- 주소 표기법: 8개의 16비트 블록을 콜론(:)으로 구분 (예: 2001:0db8:85a3:0000:0000:8a2e:0370:7334)

#### IPv6의 장단점

- 장점:
  - 거의 무제한의 IP 주소 제공
  - 향상된 보안 기능 (IPsec 내장)
  - 효율적인 라우팅
- 단점:
  - IPv4와의 호환성 문제
  - 기존 네트워크 장비 교체 필요

많은 이동통신사와 대형 기업들이 IPv6를 도입하고 있습니다. 예를 들어, Google은 자사의 모든 서비스에 대해 IPv6 지원을 제공하고 있습니다. 한국에서는 아직 적극적으로 도입되지는 않은 편입니다.
