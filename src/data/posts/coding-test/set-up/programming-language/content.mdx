## 코딩 테스트 언어 선택

파이썬은 문법이 간단하고 수도코드(Pseudocode)를 작성한 뒤 그에 가깝게 바로 구현할 수 있는 간단한 언어입니다. 변수 자료형의 제약도 없고 라이브러리도 풍부한 편이라 지금까지는 주로 파이썬으로 코딩테스트를 풀어왔습니다. 많은 기업 코딩테스트에서도 파이썬을 지원하고 있어서 지금까지는 큰 불편함은 없었습니다.

이번에 Next.js 프로젝트를 새롭게 준비하면서 타입스크립트 혹은 자바스크립트로 코딩테스트를 한번 준비해봐야겠다는 생각이 들었습니다. 포스트 관련 기능을 구현하면서 조금만 더 언어에 대한 이해도를 올리고 싶은 느낌? 저는 로직을 구현할 때 막히면 일단 공식 문서를 찾아봅니다. Next, React, JavaScript는 각각 사이트 공식 문서나 MDN Docs에 잘 정리가 된 편이라 좋았습니다. 거기서도 찾기 힘들면 생성형 AI를 시켜서 초안을 만들어 본 뒤, 그걸 참고해서 공식 문서를 다시 찾아보거나 구글링해서 이론적인 내용을 보강합니다. 그래도 역시 직접 구현해본 경험에 대비해서 생산성이 좀 낮습니다. 코딩테스트 문제가 실제 구현과 다르다는 것은 알지만, 문제풀이 과정에서 얻게되는 인사이트는 확실히 있습니다.

그런 측면에서 파이썬은 확실히 디메리트가 있습니다. 파이썬으로 문제를 풀 때 저는 리스트 컴프리헨션과 range 객체, 그리고 각종 라이브러리를 가져다 쓰는 편입니다. 이를 통해 오롯이 문제 해석에만 집중하고 요구사항을 빠르게 구현할 수는 있었습니다. 하지만 제가 기능을 구현할 때 주로 쓰는 언어는 아닙니다. 그때 얻은 인사이트를 자바나 자바스크립트 로직 구현에 녹여내려고 노력하고는 있지만, 역시 언어적 차이가 있습니다.

파이썬은 언급한대로 문법이 간결하고 사용가능한 표준 라이브러리가 많습니다. 그만큼 메모리 효율과 실행 시간에 있어서 손해를 보는 편입니다. 자바는 파이썬보다는 실행 속도가 빠르지만 생각보다 많은 메모리를 사용하는듯 합니다. 표준 라이브러리에는 구글 구아바 등에서 지원하는 콜렉션을 사용할 수 없어서 불편한 부분이 있습니다. C++은 파이썬보다 더 긴 코드를 작성해야하지만, 메모리와 실행 시간에 있어서 큰 이점이 있다고 생각합니다. 자바스크립트는 프론트엔드 채용이 아니라면 큰 메리트는 없다고 생각하지만, 그냥 제가 관심이 있습니다.

## 프로그래밍 언어별 간단한 문법적 차이

문제풀이에 앞서 각 언어별 문법적 차이를 간단하게 정리해봤습니다. C++, Java, JavaScript, Python 순으로 간단한 코드 스니펫을 작성했습니다. 제네릭, 구조체, 클래스, 루프, 반복문 등 다섯 가지 주제로 비교해보았습니다.

### 1 제네릭(Generic)

#### C++

```c++
#include <iostream>

template<class T, class U>
bool are_equal(T a, U b) {
  return (a == b);
}

int main() {
  std::cout << are_equal(10, 10.0) << std::endl;
  return 0;
}
```

- C++은 템플릿을 사용해 제네릭을 구현합니다. Java의 제네릭처럼 타입을 추상화할 수 있습니다.
- 비교 함수의 매개변수 a와 b는 함수를 정의하는 시점에는 타입이 결정되지 않으며, 서로 다른 타입을 가질 수 있습니다.

#### Java

```java
public class Main {
    public static<T, U> boolean are_equal(T a, U b) {
        return a == b;
    }
    public static void main(String[] args) {
        System.err.println(are_equal(10, 10.0));
    }
}
```

- Java는 함수 시그니처에서 대괄호를 사용해 제네릭을 구현합니다.
- C++과 마찬가지로 T, U는 함수를 정의하는 시점에는 타입이 결정되지 않으며, 서로 다른 타입을 가질 수 있습니다.

#### TypeScript

```typescript
function are_equal<T, U>(a: T, b: U): boolean {
  return +a === +b;
}

console.log(are_equal(10, 10.0));
```

- JavaScript는 제네릭을 지원하지 않습니다. TypeScript를 사용하면 제네릭을 사용할 수 있습니다.
- TypeScript는 함수 시그니처에서 대괄호를 사용해 제네릭을 구현합니다.

#### Python

```python
from typing import TypeVar

T = TypeVar('T')
U = TypeVar('U')

def are_equal(a: T, b: U) -> bool:
    return a == b

print(are_equal(10, 10.0))
```

- Python은 동적 타이핑(Dynamic Typing) 언어라서 제네릭이 필요 없습니다. 변수는 모든 타입이 될 수 있습니다. 하지만 타입스크립트가 도입된 이유와 마찬가지의 이유로 타입 힌트가 도입되었습니다.

### 2 구조체(Struct)

#### C++

```c++
#include <iostream>

struct Product {
  int weight;
  double price;
};

int main() {
  Product product;
  product.weight = 150;

  std::cout << "Weight: " << product.weight << std::endl;
  std::cout << "Price: " << product.price << std::endl;
  return 0;
}
```

- C++은 `struct` 키워드를 사용해 구조체를 정의합니다. 구조체는 메모리에 할당 가능한 복합 자료형입니다. 클래스와 비슷하지만, 클래스보다 이전부터 존재한 문법입니다.

#### Java

```java
class Product {
    private int weight;
    private double price;

    public int getWeight() {
        return weight;
    }

    public void setWeight(int weight) {
        this.weight = weight;
    }

    public double getPrice() {
        return price;
    }

    public void setPrice(double price) {
        this.price = price;
    }
}

public class Main {
    public static void main(String[] args) {
        Product product = new Product();
        product.setWeight(100);

        System.out.println("Product weight: " + product.getWeight());
        System.out.println("Product price: " + product.getPrice());

    }
}
```

- Java는 더 이상 구조체를 지원하지 않습니다. 그래서 유사한 기능을 구현하기 위해서는 클래스를 사용해야 합니다.

#### TypeScript

```typescript
interface Product {
  weight: number;
  price: number;
}

let product = {} as Product;
product.weight = 150;

console.log(product.weight);
console.log(product.price);
```

- 타입스크립트도 구조체를 지원하지 않습니다. 구조체의 목적이 간단한 데이터 그룹화에 있다는 걸 생각하면, 인터페이스를 사용해 비슷한 개념을 구현할 수 있습니다. 하지만 구조체는 아닙니다.

#### Python

```python
from dataclasses import dataclass

@dataclass
class Product:
    weight: int = None
    price: float = None

product = Product()
product.weight = 150

print(product.weight)
print(product.price)
```

- 파이썬도 구조체를 지원하지 않습니다. 파이썬 3.7+ 부터 `dataclass` 데코레이터를 사용해 데이터 클래스를 정의할 수 있습니다. 데이터 클래스는 데이터 보관과 관련된 기능이 추가되어 있습니다. 하지만 구조체는 아닙니다.

### 3 클래스(Class)

#### C++

```c++
#include <iostream>

// 선언부
class Rectangle {
  int width, height;
  public:
    Rectangle(int, int);

    int area();
};

// 구현부
Rectangle::Rectangle(int w, int h) {
  width = w;
  height = h;
}

// 구현부
int Rectangle::area() {
  return width * height;
}

int main() {
  Rectangle rectangle(3, 4);
  std::cout << "Area: " << rectangle.area() << std::endl;
  return 0;
}
```

- C++은 클래스를 정의할 때 `class` 키워드를 사용합니다. 선언부와 구현부를 분리해서 작성할 수 있습니다.
- 클래스의 멤버 변수와 멤버 함수는 기본적으로 `private`으로 선언됩니다. `public` 키워드를 사용해 외부에서 접근 가능하도록 변경할 수 있습니다.
- 클래스의 멤버 함수를 정의할 때 `::` 연산자를 사용해 클래스 이름과 함수 이름을 구분합니다.
- 클래스의 생성자는 클래스 이름과 동일한 이름을 가집니다. new 연산자없이 바로 객체를 생성할 수 있습니다.

#### Java

```java
class Rectangle {
    int width;
    int height;

    public Rectangle(int width, int height) {
        this.width = width;
        this.height = height;
    }

    public int area() {
        return this.width * this.height;
    }
}

public class Main {
    public static void main(String[] args) {
        Rectangle rectangle = new Rectangle(3, 4);
        System.out.println("Area: " + rectangle.area());
    }
}
```

- Java는 클래스를 정의할 때 `class` 키워드를 사용합니다. 클래스 안에 멤버 변수와 멤버 함수를 정의할 수 있습니다.
- 접근제어자 `default`는 같은 패키지 내에서만 접근 가능합니다.
- 객체를 생성할 때 `new` 연산자를 사용합니다.

#### TypeScript

```typescript
class Rectangle {
  width: number;
  height: number;

  constructor(width: number, height: number) {
    this.width = width;
    this.height = height;
  }

  area(): number {
    return this.width * this.height;
  }
}

const rectangle = new Rectangle(3, 4);
console.log(rectangle.area());
```

- TypeScript는 클래스를 정의할 때 `class` 키워드를 사용합니다. 클래스 안에 멤버 변수와 멤버 함수를 정의할 수 있습니다.
- 자바스크립트의 클래스는 프로토타입의 상속을 사용해 클래스와 유사한 기능을 제공합니다. 엄밀히는 클래스가 아닙니다.
- 기본적으로 `public` 접근제어자를 사용합니다.
- 객체를 생성할 때 `new` 연산자를 사용합니다.

#### Python

```python
from dataclasses import dataclass

@dataclass
class Rectangle:
    width: int
    height: int

    def area(self):
        return self.width * self.height

rectangle = Rectangle(3, 4)
print(rectangle.area())
```

- 파이썬은 클래스를 정의할 때 `class` 키워드를 사용합니다. 클래스 안에 멤버 변수와 멤버 함수를 정의할 수 있습니다.
- 클래스의 생성자는 `__init__` 메서드로 정의합니다. `self`는 인스턴스 자신을 가리킵니다.
- dataclass 데코레이터를 사용하면 Spring의 Lombok처럼 생성자, `__str__`, `__eq__` 등의 메서드를 자동으로 생성할 수 있습니다.

### 4 루프(Loop)

#### C++

```c++
#include <iostream>

int main() {
  int sum = 0;
  for (int i = 1; i <= 10; i++) {
    sum += i;
  }

  std::cout << "Sum: " << sum << std::endl;
  return 0;
}
```

- C++의 프로그램 진입점(entry point)는 `int main() { }` 함수입니다. Java의 `public static void main(String[] args) { }`와 같은 역할입니다. 반환 타입이 int이기 때문에 0을 반환해 프로그램이 정상적으로 종료되었음을 알립니다.
- `int` 타입의 `sum` 변수 선언이나, `for` 루프의 사용은 Java와 동일합니다.
- C++은 `std::cout`을 사용해 콘솔에 출력합니다. Java의 `System.out.println()`과 비슷한 역할입니다. `<<` 연산자를 사용해 출력 스트림에 데이터를 전달합니다. `std::endl`은 출력 스트림을 비우고 줄바꿈을 수행합니다. Java의 `System.out.println()`과 같은 역할입니다.
- C++은 출력 스트림을 사용하기 위해 `#include <iostream>` 헤더 파일을 포함해야 합니다. Java의 `import`와 같은 역할입니다.

#### Java

```java
public class Main {
    public static void main(String[] args) {
        int sum = 0;
        for (int i = 1; i <= 10; i++) {
            sum += i;
        }

        System.out.println(sum);
    }
}
```

- Java의 클래스명은 대문자로 시작합니다. 그리고 파일명과 클래스명이 일치해야 합니다.
- Java의 프로그램 진입점(entry point)은 `public static void main(String[] args) { }` 메서드입니다. C++의 `int main() { }`과 같은 역할입니다. 반환 타입이 `void`이기 때문에 아무것도 반환하지 않습니다. 문자열 배열 `args`를 인자로 받습니다.
- `int` 타입의 `sum` 변수 선언이나, `for` 루프의 사용은 C++과 동일합니다.
- Java은 표준 출력에 `System.out.println()`을 사용합니다.

#### TypeScript

```typescript
let sum = 0;
for (let i = 1; i <= 10; i++) {
  sum += i;
}

console.log(sum);
```

- React를 개발할 때는 let의 사용을 피해왔지만, 코딩 테스트에서는 재할당 가능한 변수인 let이 필요합니다.
- 루프의 표현은 C++ 및 Java와 비슷합니다. 정확히는 C의 문법과 비슷합니다.
- TypeScript는 표준 출력에 `console.log()` 함수를 사용합니다.

#### Python

```python
sum = sum(range(1, 11))

print(sum)
```

- 파이썬은 일반적인 루프, 리스트 컴프리헨션, range 객체 등 다양한 방법으로 루프를 표현할 수 있습니다.
- `range(1, 11)`은 1부터 10까지의 숫자를 생성합니다. `sum()` 함수는 반복가능한 객체의 요소를 모두 더한 값을 반환합니다.
- 함수명과 같은 변수 `sum`을 사용할 수도 있고, 그 안에 `sum()` 함수의 반환값을 담을 수도 있습니다.
- 파이썬은 표준 출력에 `print()` 함수를 사용합니다.

### 5 반복문(Iteration)

#### C++

```c++
#include <iostream>

int main() {
  std::string foo[] = {"A", "B", "C"};
  for (std::string bar : foo) {
    std::cout << bar << std::endl;
  }
  return 0;
}
```

- C++은 범위 기반 for 루프를 사용해 배열의 요소를 순회합니다. Java의 for-each 루프와 비슷합니다.

#### Java

```java
public class Main {
    public static void main(String[] args) {
        String[] foo = {"A", "B", "C"};
        for (String bar : foo) {
            System.out.println(bar);
        }
    }
}
```

- Java는 for-each 루프를 사용해 배열의 요소를 순회합니다. 향상된 for 문이라고도 불립니다.

#### TypeScript

```typescript
let foo: string[] = ['A', 'B', 'C'];
for (const bar of foo) {
  console.log(bar);
}
```

- TypeScript는 for of 루프를 사용해 배열의 요소를 순회합니다.

#### Python

```python
for bar in ["A", "B", "C"]:
    print(bar)
```

- Python은 for in 루프를 사용해 배열의 요소를 순회합니다.
